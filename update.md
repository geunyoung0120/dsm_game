# 업데이트 기록

이 파일은 지금까지 구현한 내용과 앞으로 추가되는 업데이트를 정리하는 문서다. 새 기능을 만들거나 기존 기능을 바꾸면 이 파일도 함께 갱신한다.

## 현재까지 구현된 내용

- Node.js, Express, Socket.io 기반 실시간 1v1 서버 구성
- SQLite 기반 사용자 DB
- 이름/비밀번호 회원가입과 로그인
- bcrypt 비밀번호 해시 저장
- 세션 쿠키 기반 로그인 유지
- 로그인 전 게임 접근 차단
- 메인 화면의 사용자 이름, 트로피, 티어 표시
- 방 만들기와 방 참가 로비
- 방 이름과 선택 비밀번호
- 대기 방 실시간 목록
- 방에 2명 입장 시 자동 경기 시작
- 경기 종료 후 트로피와 티어 자동 갱신
- 킹타워 파괴 승리 +2 트로피
- 타워 HP 우위 승리 +1 트로피
- 패배 2회마다 -1 트로피
- 트로피 0 하한과 마이어스~챔피언 티어
- Phaser.js 기반 게임 화면 렌더링
- 방의 첫 번째 접속자 아래 진영, 두 번째 접속자 위 진영 자동 배정
- 킹타워 1개와 프린세스타워 2개씩 배치
- 타워 자동 공격
- 킹타워 파괴 시 즉시 승리
- 프린세스타워 파괴 전 킹타워 공격 제한
- 3분 제한 시간
- 정규 시간 종료 시 타워 총 HP 비교 승리
- HP 동률 시 서든 데스 진입
- 최대 10 엘릭서 자동 충전
- 마지막 1분 타이머 아래 `X2` 표시
- X2 상태에서 엘릭서 2배 충전
- 12장 카드풀
- 각 플레이어가 경기 시작 시 12장 중 8장을 독립적으로 무작위 추첨
- 4장 손패 표시
- 카드 사용 후 다음 카드 순환
- 일반 카드는 8장 사용 후 처음 카드부터 다시 순환
- 덱 총합 코스트 40~55 검증
- 꽁호와 대.근.영은 사용 후 덱에서 제거되는 1회용 카드
- 카드별 코스트와 기본 전투 스탯 적용
- 자기 진영 안에서만 카드 소환 가능
- 꽁호 1회용 필드 리셋
- 빼트맨의 여학생 카드 자동 추적 힐
- 박바둑 랜덤 광역 공격과 아군 피해
- 유신 8명 동시 소환
- 지민 공격 준비 동작과 피격 시 공격 취소
- 미토스건휘 HP 50% 이하 자동 각성
- 미토스건휘 각성 중 무적
- 복숭아 근접 연타 유닛
- 성주 원거리 키보드 공격 유닛
- 조현우 근접 단일 폭딜 유닛
- 대.근.영 전장 생존 중 광주 탱크 자동 소환과 킥복싱 공격
- 김루이 부착 흡혈과 대상 행동 봉쇄
- 메인 화면의 `게임 시작` 버튼
- 메인 화면의 `캐릭터 확인` 버튼
- 12장 캐릭터 도감 페이지와 뒤로가기 버튼
- 상대 손패 숨김
- Railway 배포용 `/healthz` 상태 확인 경로
- 보안 헤더와 CSP 적용
- HTTP 요청 및 Socket.io 접속 속도 제한
- 비정상 소켓 이벤트 반복 시 연결 차단
- Socket.io Origin 검사
- 소환, 공격, 힐, 각성, 승천, 서든 데스 시각 효과
- 꽁호 승천의 찬송음, 중앙 기도 실루엣, 유닛별 반응 문구
- 기본 smoke 테스트 스크립트 구성

## 2026-06-24

- SQLite 사용자 DB를 추가하고 사용자 이름, 비밀번호 해시, 트로피, 티어, 패배 카운터를 저장하도록 했다.
- 회원가입과 로그인 API를 추가하고 bcrypt 해시와 세션 쿠키로 인증을 처리하도록 했다.
- 로그인하지 않은 Socket.io 연결을 거부해 게임 상태와 방 기능에 접근하지 못하게 했다.
- 메인 화면에 사용자 이름, 트로피, 티어를 표시하도록 했다.
- 방 만들기, 방 참가, 선택 비밀번호, 실시간 대기 방 목록을 추가했다.
- 전역 단일 경기 상태를 방별 경기 상태로 바꿔 각 방이 독립적으로 전투를 진행하도록 했다.
- 한 방에 두 명이 들어오면 자동으로 경기가 시작되도록 했다.
- 경기 종료 후 킹타워 파괴 승리는 +2, HP 우위 승리는 +1 트로피를 지급하도록 했다.
- 패배 2회마다 트로피가 1 감소하고 0 아래로 내려가지 않도록 했다.
- 트로피 수에 따라 마이어스, 브론즈, 실버, 골드, 다이아몬드, 마스터, 챔피언 티어가 자동 갱신되도록 했다.
- HUD에 양쪽 플레이어의 이름, 티어, 트로피, 타워 체력을 표시하도록 했다.
- 경기 종료 화면에 내 트로피 변동과 현재 티어를 표시하도록 했다.
- 꽁호 승천 연출에 찬송음, 중앙 기도 실루엣, 유닛별 반응 문구를 추가했다.
- smoke 테스트를 회원가입, 인증 소켓 연결, 방 생성, 방 참가, 카드 사용 흐름으로 갱신했다.
- Railway 배포를 고려해 보안 헤더, CSP, iframe 차단, MIME 스니핑 차단을 추가했다.
- `ALLOWED_ORIGINS` 또는 같은 Host 기준으로 Socket.io 접속 출처를 검사하도록 했다.
- HTTP 요청과 Socket.io 연결 요청에 속도 제한을 추가했다.
- 비정상 `play-card` 이벤트가 반복되면 소켓을 끊도록 했다.
- 경기 중 악의적인 `restart` 이벤트를 막았다.
- 상태 전송을 플레이어별로 나눠 상대 손패가 클라이언트에 노출되지 않도록 했다.
- 도감 렌더링에서 HTML 문자열 삽입을 없애고 DOM API로 생성하도록 바꿨다.
- Railway 헬스체크용 `/healthz` 경로를 추가했다.
- 12장 카드풀을 기준으로 각 플레이어가 8장 랜덤 덱을 독립적으로 뽑도록 수정했다.
- 덱 총합 코스트가 40~55가 아니면 다시 뽑도록 했다.
- 일반 카드는 사용 후 다시 순환하고, 꽁호와 대.근.영은 사용 후 덱에서 제거되도록 했다.
- 성주, 조현우, 대.근.영, 김루이를 카드풀에 추가했다.
- 대.근.영의 탱커 하수인 자동 소환을 구현했다.
- 김루이의 부착, 흡혈, 자가 회복, 대상 묶임, 피격 시 이탈을 구현했다.
- 메인 화면에 `게임 시작`과 `캐릭터 확인` 버튼을 추가했다.
- 12명 캐릭터의 비용, 타입, 능력, 외형, 특징을 보여주는 캐릭터 도감 페이지를 추가했다.
- 게임 안에서 보이는 영어 표기를 한국어로 정리했다.
- 랜덤 손패에 맞춰 smoke 테스트가 엘릭서 충전을 기다릴 수 있게 수정했다.
- 정규 경기 마지막 1분부터 `X2`가 표시되도록 HUD를 수정했다.
- `X2` 상태에서 엘릭서 충전 속도가 2배가 되도록 서버 충전 로직을 수정했다.
- `game_design_document.md`를 `agent.me`로 바꾸고 게임 룰과 구현 방식 중심 문서로 정리했다.
- `README.md`를 프로젝트 기술 구조, 백엔드 뼈대, 프론트엔드 뼈대 중심으로 정리했다.
- `update.md`를 새로 만들고 지금까지 구현된 내용을 정리했다.
- 대.근.영의 본체 HP, 공격력, 공격 속도와 탱커 하수인의 HP, 공격력을 낮추고 탱커 소환 주기를 1초에서 1.7초로 늘려 압박력을 줄였다.
- 대.근.영의 본체 공격력을 200으로 추가 하향하고, 표시 이름을 `대.근.영`으로 바꿨다.
- 지민이 유신을 공격할 때 상대 유신 유닛 전체를 함께 공격하도록 하고, 캐릭터 확인 페이지의 지민 설명도 갱신했다.
- 캐릭터 변경 시 캐릭터 확인 페이지 설명도 항상 함께 갱신하도록 설계 메모에 문서 관리 규칙을 추가했다.
- 킹타워와 프린세스타워 체력을 2배로 늘리고, 타워 공격력은 약 20% 낮췄다.
- 박바둑과 조현우가 동시에 손패에 있으면 절친 특성으로 8 엘릭서에 둘 다 함께 출격하도록 했다.
- 캐릭터 확인 페이지의 박바둑, 조현우 설명에 절친 특성을 반영했다.
- 지민이 유신을 공격할 때 머리 위에 `인포 가위질!` 문구와 전용 임팩트가 뜨도록 했다.
- 박바둑과 조현우 절친 출격 시 전장에 `절친 특성!` 문구와 임팩트가 뜨도록 했다.
- SQLite 저장 경로가 Railway Volume의 `RAILWAY_VOLUME_MOUNT_PATH`를 자동으로 사용하도록 바꿔 배포 후 계정 데이터가 유지되게 했다.
- production에서 영구 저장 경로 없이 실행되면 계정 데이터 유실 가능성을 경고하도록 했다.
- 지민의 공격 주기와 드립 준비 시간을 절반으로 줄여 공격 체감 속도를 약 2배 빠르게 했다.
- 캐릭터 피해량을 전체적으로 약 20% 낮추고, 유신 HP를 절반으로 낮췄다.
- 빼트맨 소환 비용을 3에서 2로 낮추고, 김루이 흡혈 피해를 추가로 낮췄다.
- 경기 종료 후 두 플레이어가 모두 동의하면 같은 방에서 재경기할 수 있는 버튼과 서버 동의 처리를 추가했다.
- 방에 있던 두 플레이어가 모두 나가면 방이 자동 삭제되도록 기존 방 정리 흐름을 재경기 상태와 맞췄다.
- 경기 중 이탈로 종료된 방도 남은 플레이어가 나가면 연결 인원 기준으로 즉시 삭제되도록 했다.
- 트로피 순 전체 랭킹 API를 추가하고, 로비 상단에 1~3등 닉네임과 티어를 표시하도록 했다.
- 로비 랭킹의 `더 보기` 버튼으로 전체 랭킹 화면을 볼 수 있게 했다.
- 대.근.영의 소환수 이름을 `광주 탱크`로 바꾸고 HP를 40% 낮췄다.
- 대.근.영이 이동 중일 때만이 아니라 전장에 있는 동안 항상 2초마다 광주 탱크를 소환하도록 바꿨다.
- 광주 탱크의 전장 표시를 사람형이 아니라 차체, 포탑, 포신, 궤도가 있는 탱크 형태로 바꿨다.

# 야클

웹 브라우저에서 실행되는 실시간 타워 디펜스 게임입니다. 백엔드가 1대1/2대2/토너먼트 경기 상태를 관리하고, 프론트엔드는 서버에서 받은 상태를 Phaser.js로 그립니다.

## 실행

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000`을 열면 됩니다. 회원가입 또는 로그인 후 메인 화면에서 방을 만들거나 대기 중인 방에 참가할 수 있습니다. 모드에 맞는 인원이 모이면 경기가 자동으로 시작됩니다.

클라이언트 전역 참조 오류를 빠르게 확인하려면 아래 명령을 실행합니다.

```bash
npm run check:client
```

## 사용 기술

| 영역 | 사용한 것 |
|---|---|
| 백엔드 언어 | JavaScript |
| 백엔드 런타임 | Node.js |
| 웹 서버 | Express |
| 실시간 통신 | Socket.io |
| 데이터베이스 | SQLite (`better-sqlite3`) |
| 인증 | Express Session, bcrypt 해시 |
| 프론트엔드 언어 | JavaScript |
| 게임 렌더링 | Phaser.js |
| 화면 스타일 | HTML, CSS |

## 백엔드 뼈대

백엔드는 `server.js` 하나를 중심으로 구성되어 있습니다.

- Express로 `public` 폴더를 정적 파일로 제공합니다.
- SQLite에 사용자 이름, 비밀번호 해시, 트로피, 티어, 패배 카운터, 토너먼트 우승 횟수를 저장합니다.
- SQLite 세션 저장소와 세션 쿠키로 로그인 상태를 유지하고, 인증되지 않은 Socket.io 접속은 거부합니다.
- Socket.io 서버를 HTTP 서버에 붙여 실시간 이벤트를 처리합니다.
- 방마다 모드에 맞춰 플레이어 슬롯과 팀을 배정합니다.
- 1대1은 두 명, 2대2는 네 명이 접속하면 `startMatch(room)`으로 경기를 초기화합니다.
- 토너먼트 방은 참가 인원과 트로피 참가비를 DB에 저장하고, 참가 시 즉시 참가비를 차감한 뒤 대진표를 생성합니다.
- 토너먼트 경기에서는 일반 트로피 승패 규칙을 적용하지 않고, 결승 종료 시 참가비 풀을 1등 70%, 2등 30%로 나눕니다.
- 토너먼트 참가비가 부족한 사용자는 확인 후 빚을 지고 참가할 수 있으며, 이때만 트로피가 마이너스로 내려갈 수 있습니다.
- 토너먼트 대진표와 진행 중 경기 목록은 Socket.io로 실시간 갱신되며, 토너먼트 관전 경기는 일반 경기보다 위에 표시됩니다.
- 완료된 토너먼트는 히스토리와 상세 대진 결과를 DB에서 다시 조회할 수 있습니다.
- 라운드 사이에는 준비시간을 두고, 결승 직전에는 2분 준비시간과 결승 전용 10장 덱 편집을 제공합니다.
- 결승전은 5분 경기, 5장 손패, 타워 HP 1.5배, X2/X3 엘릭서 타이밍 변경을 적용합니다.
- 서버 안의 방 목록이 각 방의 경기 상태, 플레이어, 타워, 유닛, 타이머, 승패 상태를 들고 있습니다.
- 50ms 간격 서버 틱에서 엘릭서 충전, 유닛 이동/공격, 타워 공격, 승리 조건을 계산합니다.
- 100ms 간격으로 현재 경기 상태를 클라이언트에 전송합니다.
- 각 플레이어는 자기 손패만 받고, 상대 손패는 서버 상태에서 숨겨집니다.
- 클라이언트가 카드를 사용하면 서버가 엘릭서, 손패, 소환 위치를 검증한 뒤 유닛을 생성합니다.
- 경기 종료 시 승리 방식과 패배 횟수에 따라 트로피와 티어를 갱신합니다.

## 계정, 방, 티어

- 회원가입과 로그인은 이름과 비밀번호만 사용합니다.
- 비밀번호는 bcrypt 해시로 저장합니다.
- 로그인 성공 후 메인 화면에 이름, 트로피, 티어가 표시됩니다.
- 프로필에는 토너먼트 우승 횟수가 표시되고, 메인 화면 최상단에는 최근 토너먼트 우승자와 우승 날짜가 표시됩니다.
- 메인 화면의 `티어표 보기` 버튼으로 전체 티어 기준과 내 현재 위치를 확인할 수 있습니다.
- 우측 상단 테마 버튼으로 라이트 모드와 다크 모드를 전환할 수 있습니다.
- 방은 이름과 선택 비밀번호를 가질 수 있습니다.
- 대기 중인 방 목록은 실시간으로 갱신됩니다.
- 킹타워 파괴 승리는 트로피 `+2`, 타워 HP 우위 승리는 트로피 `+1`입니다.
- 패배는 2번마다 트로피 `-1`이며, 트로피는 0 아래로 내려가지 않습니다.
- 티어는 트로피에 따라 마이어스, 브론즈, 실버, 골드, 다이아몬드, 마스터, 챔피언으로 자동 갱신됩니다.

## Railway 배포 설정

Railway는 `PORT` 환경 변수를 자동으로 넣어주므로 별도 포트 설정 없이 `npm start`로 실행하면 됩니다.

계정과 세션은 SQLite 파일에 저장됩니다. Railway에서 배포할 때 계정을 유지하려면 서비스에 Volume을 붙여야 합니다. Volume이 붙으면 Railway가 `RAILWAY_VOLUME_MOUNT_PATH`를 자동으로 제공하고, 서버는 그 경로의 `game.sqlite`를 사용합니다. Volume이 없으면 배포 때 앱 파일시스템이 새로 만들어져 기존 계정이 사라질 수 있습니다.

보안 강화를 위해 Railway 환경 변수에 아래 값을 추가하는 것을 권장합니다.

```bash
ALLOWED_ORIGINS=https://배포된-도메인
SESSION_SECRET=충분히-긴-랜덤-문자열
NODE_ENV=production
```

직접 경로를 지정하려면 아래 환경 변수 중 하나를 추가하면 됩니다.

```bash
DATA_DIR=/app/data
# 또는
DB_PATH=/app/data/game.sqlite
```

`ALLOWED_ORIGINS`를 설정하면 해당 도메인에서 온 Socket.io 접속만 허용합니다. 여러 도메인을 허용해야 하면 쉼표로 구분합니다.

서버 상태 확인 경로:

```text
/healthz
```

## 보안 처리

- 기본 보안 헤더와 CSP를 적용합니다.
- iframe 삽입을 차단합니다.
- MIME 타입 스니핑을 차단합니다.
- 비밀번호는 평문 저장 없이 bcrypt 해시로 저장합니다.
- 로그인 세션은 HttpOnly 쿠키와 SQLite 세션 저장소로 유지합니다.
- HTTP 요청과 Socket.io 접속에 간단한 속도 제한을 적용합니다.
- 비정상 `play-card` 이벤트가 반복되면 해당 소켓을 끊습니다.
- 경기 중 악의적인 `restart` 이벤트는 무시하고 누적 위반으로 처리합니다.
- 상대 손패는 클라이언트로 보내지 않습니다.

## 프론트엔드 뼈대

프론트엔드는 `public` 폴더 안에 있습니다.

- `public/index.html`은 Phaser와 Socket.io 클라이언트 스크립트를 불러옵니다.
- `public/styles.css`는 브라우저 화면 중앙에 게임 캔버스를 배치합니다.
- `public/src/main.js`는 Phaser `BattleScene`으로 게임 화면을 그립니다.
- 로그인, 회원가입, 메인 프로필, 방 생성/참가 화면을 DOM으로 관리합니다.
- 클라이언트는 서버의 `state` 이벤트를 받아 타워, 유닛, HUD, 카드 UI를 갱신합니다.
- 카드를 클릭하고 자기 진영을 클릭하면 `play-card` 이벤트를 서버로 보냅니다.
- 서버의 `effect` 이벤트를 받아 공격, 소환, 각성, 승천 같은 시각 효과를 표시합니다.
